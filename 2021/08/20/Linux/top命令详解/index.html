top命令详解

发表于 2021-08-20 更新于 2021-08-21 分类于 Linux

top 第一行：

系统时间 系统自上次重启运行了多长时间 当前登录用户数

系统负载：最近1分钟，最近5分钟，最近15分钟。监控数据一般会使用最近1分钟的指标。

top第二行：

总任务数 运行状态任务数，睡眠状态任务数，终止状态任务数，僵尸状态任务数。

当有僵尸任务时，可以查看状态为Z的任务，或者使用ps aux|grep Z过滤。

top第三行：

us:高优先级用户态进程占用cpu时间百分比(nice<=0的)

sy:内核态进程占用cpu时间百分比

ni:低优先级用户态进程占用cpu时间百分比(nice > 0)

id:空闲态占用cpu时间百分比–> cpu利用率100-idle

wa:等待io请求完成占用cpu时间百分比

hi:硬中断占用cpu时间百分比

si:软中断占用cpu时间百分比

st:虚拟机被hypervisor偷走时间占用cpu时间百分比

注意：这个是显示总体cpu的平均使用情况，计算时需要除以cpu的个数，如果需要显示单个cpu的事情情况，可以按键1，下面表示系统有两个cpu，cpu占用繁忙。

top第四行：

KiB Mem : 3848656 total, 135212 free, 2260244 used, 1453200 buff/cache
单位为KB 总物理内存 空闲内存 被使用的内存 被缓存的内存

top第五行：

单位为KB 总虚拟内存 空闲虚拟内存 被使用的虚拟内存 可使用物理内存

总虚拟内存为0，说明系统没有开启swap。

从这里得知可使用物理内存为1308928KB，内存利用率=(3848656-1308928)/3848656。

top 最后一行：

PID:进程id

USER:进程用户

PR:任务的动态调度优先级，取值范围是[-100,39]

NI:任务的静态调度优先级，通过nice命令修改，值越大，优先级越低，取值范围是[-20,19]

VIRT:任务使用虚拟内存大小，单位是kb，包含代码、库以及交换出去的page等，没啥参考价值

RES:任务使用物理内存大小，单位是kb

SHR:任务使用共享内存大小，单位是kb

S:任务的状态，可以为以下值：

  • ’D’ = uninterruptible sleep

  • ’R’ = running

  • ’S’ = sleeping

  • ’T’ = traced or stopped

  • ’Z’ = zombie

%CPU:任务使用cpu时间百分比，使用多核时是相加起来，所以值会大于100%

%MEM:物理内存使用率，RES/total

TIME+:任务使用的cpu时间，单位为秒，精确到百分之秒

COMMAND:启动任务的命令行或者关联程序名字

关于PR和NI的区别

PR和NI都表示优先级，但是含义有些不同。在操作系统中，进程分为两种类型：用户进程和实时进程。进程的优先级有140个级别。

用户进程：

nice命令就是修改的用户进程的优先级，表示的是NI的值，范围是[-20,19]，此时PR=20+NI，它对应的就是100-139的优先级，优先级较低。

实时进程：

chrt命令就是修改实时进程的优先级，此时PR=-1-real_time_priority，实时优先级取值范围是[1,99]，它对应的就是0-99的优先级，优先级较高。
